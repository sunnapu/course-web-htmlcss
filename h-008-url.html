http://www.example.com:80/posts?name=john&age=15#section1

  • protocol: http
  • host: www.example.com
  • port: 80
  • path: /posts
  • query: ?name=john&age=15
  • fragment url: #section1 (a location within a resource, this will not be sent to server)

URL and URI and URN

  • People use them interchangably, but not correct
  • URI: Uniform Resource Identifier, identify a resource by location or a name or both. It can be URL and URN more specifically, but URL is not always URI
  • URL: Uniform Resource Locator, specify where a resource is and how to access it, such as http://domain.com, ftp://, smb:// etc
  • URN: Uniform resource name
  • example of URI: 
          ftp://ftp.is.co.za/rfc/rfc1808.txt
