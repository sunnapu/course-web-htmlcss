Why use background-attachment?

use background-attachment to control local view scroll behaviors

the local view is a div in the web page (main view)

In this example, background image is set as the local view (in a div), the local view is in the main view (page).

scroll (default): background scrolls when the main view scrolls, but doesn't scroll when the local view scrolls.

local: background scrolls when either the main or local view scrolls

fixed: background doesn't scroll no matter what (unless you move monitor)

    The following steps make main view and local view scroll:
    • main view (page) height larger than viewport, so it can scroll up and down, in this case, by body padding-bottom
    • the 'expand' height larger than its container div, so it can scroll up and down
    • the 'expand' has margin to its container div, so it can scroll left and right
